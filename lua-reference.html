luaref.txt
==========

The gigantic reference that gives you knowledge on literally everything that you
can do with The Powder Toy Lua!

Notes
-----

* The tree below is pretty packed with information. The doc has long lines so it
  is suggested that you disable word wrap.
* boolint = integer that acts as a boolean (0 or 1)
* number = Lua's natural number type, = "real"
* I'll insert default values or other notes right next to the argument inside 
  brackets []. 
* d. is short for default. 
* 1||2 in tpt.setwindowsize means that the correct value is either 1 or 2, 
  nothing else.
* 0:255 means a value between 0 and 255.
* mixed type means "whatever fits there"
* NYI: Not Yet Implemented
* If not explicitly noted, all color values have limits of 0 to 255 (incl. firer/g/b, colr/g/b) and default to white.
* Extension to the above: all alpha values (firea/cola) have a limit of 100 as the most opaque.
* Did you know every section has its shortname as its anchor and
  every property has a section.property anchor? You can link to everything here!
  For example: tptelements/lua-reference.html#elem.PROP_NOCTYPEDRAW

Properties

An element has many properties as seen above in the tpt.parts[] array definition. 
I've copied the list below for reference.

Elements
~~~~~~~~

* Note: mul stands for multiplier. Used to determine the end result by
  multiplying it to the determined one before application.
* A range [0:x] without a second element [0:] means it can go on however much.

tpt.el.[inst].*
  │
  ├─ char[4] name        -- short name - must be 4 characters.
  ├─ int color           -- element's default colour. 0xAARRGGBB (Element menu uses 0xRRGGBB## !!)
  ├─ float advection     -- how much moving air pushes the particle [d. 1f:0f] 
  ├─ float airdrag       -- how much air this particle 'drags' with it when moving[d. 1f:0f] 
  ├─ float airloss       -- how much it slows down moving air nearby [d. 1f:0f] 
  ├─ float loss          -- how much speed this particle loses every frame [d. 1f:0f] 
  ├─ float collision     -- own velocity is multiplied by this on collision [mul: 0f]
  ├─ float gravity       -- how fast said particle drops down - ANAR has negative values [signed: 0f] 
  ├─ float diffusion     -- how much the particle 'wiggles' around like OXYG [mul: 0f]
  ├─ float hotair        -- how much the particle increases surrounding pressure by - for gas diffusion [mul: 0f]
  ├─ int falldown        -- how and whether this particle falls down [0:solid||1:powder||2:liquid]
  ├─ int flammable       -- how well this particle burns [d. 0f:]
  ├─ int explosive       -- how and whether this particle explodes [0:no||1:fire||2:fire/2.5pressure]
  ├─ boolint meltable    -- how and whether this particle melts [0:no|1:yes]
  ├─ int hardness        -- how well this particle corrodes via acid - a multiplier [d. 0f:]
  ├─ boolint menu        -- whether or not this element is visible in a menu
  ├─ boolint enabled     -- whether or not this element is even usable
  ├─ int weight          -- how heavy this element is (1 = gas. 2 = light. 0-49 = liquids. 50-99 = powders. 100 = solid. -1 for energy.)
  ├─ int menusection     -- which menu section said element resides in [refer to]
  ├─ float heat          -- which heat the element spawns with when created [0f:10000f] in Kelvin (Celsius-273.15)
  ├─ int hconduct        -- how well this element juggles around heat [0:250]
  ├─ int state           -- element state [refer to]
  ├─ int properties      -- various quirks of an element [refer to]
  └─ char * description  -- a string of text for the element's description
